# ESP32 Art-Net → ESP-NOW Master Node

Прошивка для **мастер-устройства** на базе ESP32 + Ethernet (W5500/W5100).  
Принимает DMX-данные по протоколу **Art-Net** через Ethernet и пересылает их слейв-нодам по **ESP-NOW** (WiFi).

---

## Архитектура системы

```
[Световое ПО]
     ↓ Art-Net (UDP, порт 6454)
  [Ethernet]
     ↓
[ESP32 MASTER]  ←→  ESP-NOW (WiFi 2.4GHz)  ←→  [ESP32 SLAVE 1]
                                             ←→  [ESP32 SLAVE 2]
                                             ←→  [ESP32 SLAVE N]
                                                       ↓
                                                  DMX (UART)
                                                       ↓
                                                  [Приборы]
```

## Схема питания

> Файл схемы: [`docs/esp32_w5500_power.drawio.svg`](docs/esp32_w5500_power.drawio.svg)

### Принципиальная схема

```
+5V IN ──► [F1 1A] ──► [D1 1N5819] ──┬──► [LM2596 5V→3.3V] ──► +3.3V ─────────────────┐
                                      │                                                   │
                                     [C1                                                  │
                                    100мкФ]                              ┌────────────────┼───────────────────┐
                                      │                         [C2]    [C3]   [C4] [C5]  │     [C6] [C7]     │
                                      │                        220мкФ  47мкФ  10мкФ 100нФ │    10мкФ 100нФ    │
                                      │                                                   │                   │
                                     GND                                            [ESP32]             [W5500]──►[RJ45]
                                                                                          │
                                                              IO25─►[R1 100Ω]─►[LED1 R]  │
                                                              IO26─►[R2 100Ω]─►[LED1 G]  │
                                                              IO27─►[R3 100Ω]─►[LED1 B]  │
                                                                              [LED1 K]──GND
```

### Расшифровка элементов схемы

#### Цепь питания (5V → 3.3V)

| Обозначение | Компонент | Назначение |
|---|---|---|
| **+5V IN** | Вход питания | Источник питания 5V (USB, адаптер, БП) |
| **F1** | Предохранитель 1A | Защита от перегрузки и короткого замыкания по питанию |
| **D1** | Диод 1N5819 (Schottky, 1A 40V) | Защита от переполюсовки. Schottky выбран за малое падение напряжения (~0.3V), не перегревает цепь |
| **LM2596** | DC-DC понижающий преобразователь 5V→3.3V, Imax 3A | Эффективное (не линейное) преобразование напряжения. КПД ~80-90%, минимум тепла |

#### Конденсаторы фильтрации и развязки

| Обозначение | Номинал | Место | Назначение |
|---|---|---|---|
| **C1** | 100мкФ 16V | После D1, до LM2596 | Входной фильтр на шине 5V: подавляет броски тока при запуске |
| **C2** | 220мкФ 10V | Выход LM2596, шина 3.3V | Выходной фильтр DC-DC: стабилизирует напряжение, гасит пульсации преобразователя |
| **C3** | 47мкФ 10V | Шина 3.3V (ближе к ESP32) | Буферный конденсатор: компенсирует пиковые токи при работе WiFi/ESP-NOW (до 500мА) |
| **C4** | 10мкФ | Питание ESP32 (3V3) | Локальная развязка питания ESP32: отфильтровывает высокочастотные помехи |
| **C5** | 100нФ | Питание ESP32 (3V3) | Высокочастотная развязка ESP32: блокирует ВЧ-шум прямо у вывода питания |
| **C6** | 10мкФ | Питание W5500 (VCC) | Локальная развязка питания W5500: стабилизирует напряжение при SPI-транзакциях |
| **C7** | 100нФ | Питание W5500 (VCC) | Высокочастотная развязка W5500: блокирует ВЧ-шум от Ethernet PHY |

> **Зачем ставить пару 10мкФ + 100нФ?** Электролитический конденсатор (10мкФ) хорошо работает на низких частотах и держит запас заряда. Керамический (100нФ) имеет низкую индуктивность и эффективен на высоких частотах. Вместе они перекрывают весь диапазон помех.

#### Основные компоненты

| Обозначение | Компонент | Питание | Описание |
|---|---|---|---|
| **ESP32-WROOM-32U** | Микроконтроллер | 3.3V | Основной вычислительный модуль. Суффикс «U» означает разъём u.FL для внешней антенны — актуально при монтаже в металлический корпус |
| **W5500** | Ethernet-контроллер | 3.3V | Аппаратный TCP/IP стек + PHY. Подключается к ESP32 по SPI. Полностью разгружает ESP32 от обработки сетевого стека |
| **J1 (RJ45)** | Разъём Ethernet | — | Стандартный разъём 8P8C для подключения кабеля Ethernet |

#### Индикация (RGB LED)

| Обозначение | Компонент | Назначение |
|---|---|---|
| **LED1** | RGB светодиод (общий катод) | Индикатор состояния системы |
| **R1** | Резистор 100Ω | Токоограничитель красного канала (IO25) |
| **R2** | Резистор 100Ω | Токоограничитель зелёного канала (IO26) |
| **R3** | Резистор 100Ω | Токоограничитель синего канала (IO27) |

> Каждый канал RGB при 3.3V и 100Ω даёт ток ~20-25мА — стандарт для светодиодов. Общий катод (K) подключён к GND.

#### Шины

| Шина | Цвет на схеме | Напряжение | Описание |
|---|---|---|---|
| **+3.3V (главная шина)** | Красный | 3.3V | Питание всех цифровых компонентов |
| **GND (общая земля)** | Синий | 0V | Общий минус всех цепей |
| **SPI** | Жёлто-коричневый (пунктир) | — | Шина данных ESP32 ↔ W5500: MOSI, MISO, SCK, CS |

### Схема подключения SPI (ESP32 → W5500)

```
ESP32          W5500
IO23 (MOSI) ──► MOSI
IO19 (MISO) ◄── MISO
IO18 (SCK)  ──► SCLK
IO5  (CS)   ──► SCSn
```

## Как работает

### 1. Art-Net приём

- Слушает UDP порт `6454`
- Обрабатывает пакеты `ArtPoll` (отвечает `ArtPollReply` — нода видна в сети)
- Обрабатывает пакеты `ArtDmx` (opcode `0x5000`) — извлекает юниверс и DMX-данные

### 2. Реестр слейвов

- Слейвы регистрируются через ESP-NOW пакет `PKT_REGISTER`
- Мастер хранит `slaveRegistry` (MAC → SlaveInfo) и `universeMap` (юниверс → список слейвов)
- Если слейвов на юниверс больше одного — используется **broadcast**, иначе **unicast**

### 3. Передача DMX

- DMX-данные (512 байт) нарезаются на **чанки по 200 байт** (ограничение ESP-NOW — 250 байт на пакет)
- Каждый чанк содержит: юниверс, оффсет, длину, порядковый номер, данные
- Порядковый номер (`sequence`) защищает от приёма устаревших пакетов

### 4. Watchdog слейвов

- Каждые 2 секунды мастер шлёт `PKT_PING` всем зарегистрированным слейвам
- Если слейв не отвечал более 8 секунд — удаляется из реестра
- Удалённый пир освобождается через `esp_now_del_peer()`

## Протокол ESP-NOW пакетов

| Тип | Направление | Описание |
|---|---|---|
| `PKT_REGISTER (0x01)` | Slave → Master | Регистрация слейва, передаёт юниверс и имя |
| `PKT_ACK (0x02)` | Master → Slave | Подтверждение регистрации, передаёт MAC мастера |
| `PKT_DMX_CHUNK (0x03)` | Master → Slave | Чанк DMX-данных (макс. 200 байт) |
| `PKT_PING (0x04)` | Master → Slave | Проверка живости |
| `PKT_PONG (0x05)` | Slave → Master | Ответ на пинг |

## Известные ограничения

- Максимум **250 байт** на ESP-NOW пакет → DMX делится на чанки по 200 байт (3 пакета на юниверс)
- ESP-NOW и WiFi используют **один радиомодуль** → канал WiFi должен совпадать с `ESPNOW_BROADCAST_CHANNEL`
- Одновременно активен **только WiFi STA** (без AP)
