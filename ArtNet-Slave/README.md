# ESP32 ESP-NOW → DMX Slave Node

Прошивка для **слейв-устройства** на базе ESP32.  
Принимает DMX-данные от мастера по **ESP-NOW** и выдаёт их на физическую линию **DMX512** через UART + RS-485 трансивер.

---

## Архитектура

```
[ESP32 MASTER]
     ↓ ESP-NOW (WiFi 2.4GHz)
[ESP32 SLAVE]
     ↓ UART (250000 baud, 8N2)
  [RS-485 трансивер]  ← обязательно!
     ↓ DMX512
  [Приборы]
```

---

## Железо

| Компонент | Описание |
|---|---|
| ESP32 | Любая плата с доступом к UART |
| TTL-RS485 модуль | MAX485, SP485, SN75176 или аналог |
| Питание | 5V (преобразуется в 3.3V через LM2596) |

### Подключение RS-485

| RS-485 модуль | ESP32 | Описание |
|---|---|---|
| DI (Data In) | GPIO 17 (`DMX_TX_PIN`) | Данные из ESP32 в RS-485 |
| RO (Receiver Out) | GPIO 16 | Данные из RS-485 в ESP32 (опционально) |
| DE + RE | GPIO 4 (`DE_PIN`) | Управление направлением передачи |
| A / B | DMX XLR-3 pin 3 / 2 | Дифференциальная пара |
| GND | GND | Общая земля |

> ⚠️ **Никогда не подключайте ESP32 напрямую к DMX линии** без RS-485 трансивера — это повредит устройство.

## Схема питания

> Файл схемы: [`docs/esp32_rs485_dmx_slave.drawio.svg`](docs/esp32_rs485_dmx_slave.drawio.svg)

### Принципиальная схема

```
+5V IN ──► [F1 1A] ──► [D1 1N5819] ──┬──► [LM2596 5V→3.3V] ──► +3.3V ────────────────────┐
                                      │                                                     │
                                     [C1                                                    │
                                    100мкФ]                           ┌─────────────────────┼────────────────┐
                                      │                      [C2]    [C3]   [C4] [C5]       │    [C6] [C7]   │
                                      │                     220мкФ  47мкФ  10мкФ 100нФ      │   10мкФ 100нФ  │
                                      │                                                     │                │
                                     GND                                              [ESP32]          [RS-485]──►[XLR-3]
                                                                                            │                       │
                                                           IO25─►[R1 100Ω]─►[LED1 R]       │                       GND
                                                           IO26─►[R2 100Ω]─►[LED1 G]       │           
                                                           IO27─►[R3 100Ω]─►[LED1 B]
                                                                           [LED1 K]──GND
```

### Расшифровка элементов схемы

#### Цепь питания (5V → 3.3V)

| Обозначение | Компонент | Назначение |
|---|---|---|
| **+5V IN** | Вход питания | Источник питания 5V (USB, адаптер, БП) |
| **F1** | Предохранитель 1A | Защита от перегрузки и короткого замыкания по питанию |
| **D1** | Диод 1N5819 (Schottky, 1A 40V) | Защита от переполюсовки. Schottky выбран за малое падение напряжения (~0.3V) |
| **LM2596** | DC-DC понижающий преобразователь 5V→3.3V, Imax 3A | Эффективное преобразование напряжения, КПД ~80-90%, минимум тепла |

#### Конденсаторы фильтрации и развязки

| Обозначение | Номинал | Место | Назначение |
|---|---|---|---|
| **C1** | 100мкФ 16V | После D1, до LM2596 | Входной фильтр на шине 5V: сглаживает броски тока при запуске |
| **C2** | 220мкФ 10V | Выход LM2596, шина 3.3V | Выходной фильтр DC-DC: стабилизирует напряжение, гасит пульсации преобразователя |
| **C3** | 47мкФ 10V | Шина 3.3V (ближе к ESP32) | Буферный конденсатор: компенсирует пиковые токи при работе WiFi/ESP-NOW (до 500мА) |
| **C4** | 10мкФ | Питание ESP32 (3V3) | Локальная развязка питания ESP32: отфильтровывает низкочастотные помехи |
| **C5** | 100нФ | Питание ESP32 (3V3) | Высокочастотная развязка ESP32: блокирует ВЧ-шум прямо у вывода питания |
| **C6** | 10мкФ | Питание RS-485 (VCC) | Локальная развязка питания RS-485: стабилизирует напряжение при переключении линии |
| **C7** | 100нФ | Питание RS-485 (VCC) | Высокочастотная развязка RS-485: подавляет ВЧ-выбросы от дифференциальных переключений |

> **Зачем ставить пару 10мкФ + 100нФ?** Электролитический (10мкФ) хорошо работает на низких частотах и держит запас заряда. Керамический (100нФ) имеет низкую индуктивность и эффективен на высоких частотах. Вместе они перекрывают весь диапазон помех.

#### Основные компоненты

| Обозначение | Компонент | Питание | Описание |
|---|---|---|---|
| **ESP32-WROOM-32U** | Микроконтроллер | 3.3V | Основной вычислительный модуль. Суффикс «U» означает разъём u.FL для внешней антенны — важно при монтаже в металлический корпус |
| **TTL-RS485** | Модуль RS-485 трансивера | 3.3V | Преобразует логический UART (3.3V) в дифференциальный сигнал DMX512. Совместим с 3.3V и 5V. Аналоги: MAX485, SP485, SN75176 |
| **J1 (XLR-3)** | Разъём DMX OUT | — | Стандартный 3-пиновый XLR-разъём для вывода DMX512. Pin 1 = GND, Pin 2 = Data–, Pin 3 = Data+ |

#### Индикация (RGB LED)

| Обозначение | Компонент | Назначение |
|---|---|---|
| **LED1** | RGB светодиод (общий катод) | Индикатор состояния устройства |
| **R1** | Резистор 100Ω | Токоограничитель красного канала (IO25) |
| **R2** | Резистор 100Ω | Токоограничитель зелёного канала (IO26) |
| **R3** | Резистор 100Ω | Токоограничитель синего канала (IO27) |

> При 3.3V и 100Ω ток через каждый канал составляет ~20-25мА — стандарт для светодиодов. Общий катод (K) подключён к GND.

#### Шины

| Шина | Цвет на схеме | Напряжение | Описание |
|---|---|---|---|
| **+3.3V (главная шина)** | Красный | 3.3V | Питание всех цифровых компонентов |
| **GND (общая земля)** | Синий | 0V | Общий минус всех цепей |
| **UART** | Жёлто-коричневый (пунктир) | — | Шина данных ESP32 ↔ RS-485: TX, RX, DE |

### Схема подключения UART (ESP32 → RS-485)

```
ESP32             TTL-RS485
IO17 (TX)  ──►   DI   ──►  A/B (DMX+/-)  ──►  XLR-3 (J1)
IO16 (RX)  ◄──   RO                                │
IO4  (DE)  ──►   DE/RE                            GND

```

## Конфигурация

```cpp
#define DMX_TX_PIN       17   // GPIO для DMX выхода (UART → RS-485 DI)
#define DMX_RX_PIN       16   // GPIO приёма (RS-485 RO → ESP32)
#define DE_PIN            4   // GPIO управления направлением (DE/RE)
#define DEFAULT_UNIVERSE  0   // Юниверс по умолчанию
```

**Юниверс** сохраняется в NVS (энергонезависимая память) и переживает перезагрузку.  
Изменить юниверс можно через Serial Monitor — отправить число и нажать Enter:

```
0    → юниверс 0
1    → юниверс 1
15   → юниверс 15
```

## Как работает

### 1. Регистрация у мастера

- При старте слейв шлёт `PKT_REGISTER` broadcast в ESP-NOW
- Мастер отвечает `PKT_ACK` с указанием своего MAC
- После этого слейв общается напрямую с мастером (unicast)
- Если мастер не ответил — повтор каждые 3 секунды

### 2. Приём DMX

- Мастер присылает `PKT_DMX_CHUNK` — чанки по 200 байт
- Слейв собирает чанки в буфер `dmxBuffer[513]` по оффсету
- Защита от дублей и устаревших пакетов через `sequence`
- Пакеты чужого юниверса игнорируются

### 3. Вывод DMX512

DMX формируется программно через сырой UART:

1. **Break** — линия LOW, 92 мкс
2. **MAB** (Mark After Break) — линия HIGH, 12 мкс
3. **Start code** (0x00) + **512 байт данных** @ 250000 baud, 8N2

Частота обновления: **~44 кадра в секунду** (каждые 23 мс)

> Break генерируется через инверсию UART (`uart_set_line_inverse`). Проверьте совместимость с вашей версией ESP-IDF.

### 4. Watchdog мастера

- Каждый `PKT_PING` обновляет таймер `lastPingReceived`
- Если мастер молчит более **10 секунд** — слейв сбрасывает регистрацию и ищет мастера заново
- DMX при этом продолжает слаться с последними значениями (безопасный режим — fixture не гаснут)

## Протокол ESP-NOW пакетов

| Тип | Направление | Описание |
|---|---|---|
| `PKT_REGISTER (0x01)` | Slave → Master | Регистрация, передаёт юниверс и имя |
| `PKT_ACK (0x02)` | Master → Slave | Подтверждение, MAC мастера |
| `PKT_DMX_CHUNK (0x03)` | Master → Slave | Чанк DMX (макс. 200 байт, оффсет внутри) |
| `PKT_PING (0x04)` | Master → Slave | Проверка живости |
| `PKT_PONG (0x05)` | Slave → Master | Ответ на пинг, подтверждает юниверс |

### Структура пакета DmxChunk

```
[type: 1] [universe: 2] [offset: 2] [length: 2] [sequence: 4] [dmx: 200]
```

Для передачи 512 каналов мастер шлёт **3 чанка**:

| Чанк | offset | length |
|---|---|---|
| 1 | 0 | 200 |
| 2 | 200 | 200 |
| 3 | 400 | 112 |

## Имя устройства

Имя формируется автоматически из последних 3 байт MAC-адреса:

```
MyName = "A1B2C3"  (последние 3 байта MAC в HEX)
```

Передаётся мастеру при регистрации для идентификации в реестре.

## Известные ограничения

- Один слейв слушает **один юниверс** (512 каналов), но несколько слейвов могут быть на одном юниверсе — мастер автоматически переключается на broadcast
- ESP-NOW работает только в диапазоне **WiFi 2.4GHz**, дальность ~200м на открытом пространстве
- Канал ESP-NOW (`ESPNOW_BROADCAST_CHANNEL = 1`) должен совпадать с настройкой мастера
- Break генерируется через инверсию UART (`uart_set_line_inverse`) — проверьте совместимость с вашей версией ESP-IDF
