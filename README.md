ArtNet ESP-NOW: беспроводная DMX система
Данный проект реализует беспроводную передачу DMX512 данных с использованием ESP32 и протоколов Art-Net (по Ethernet) и ESP-NOW (Wi-Fi peer-to-peer). Система состоит из мастер-узла, который принимает Art-Net от светового ПО и рассылает DMX данные слейв-узлам по ESP-NOW, и слейв-узлов, которые преобразуют ESP-NOW пакеты в физический сигнал DMX512 через RS-485.

Архитектура системы

[Световое ПО (QLC+, FreeStyler и др.)]
        │ Art-Net (UDP, порт 6454)
        ▼
   [Ethernet]
        │
   [ESP32 MASTER]  ←→  ESP-NOW (Wi-Fi 2.4 ГГц)  ←→  [ESP32 SLAVE 1]
                                                 ←→  [ESP32 SLAVE 2]
                                                 ←→  [ESP32 SLAVE N]
                                                           │
                                                      DMX (UART)
                                                           │
                                                      [RS-485]
                                                           │
                                                      [Световые приборы]

Мастер подключается к локальной сети по Ethernet (W5500), слушает Art-Net пакеты и пересылает DMX данные слейвам по ESP-NOW.

Слейвы принимают ESP-NOW пакеты, собирают полный DMX фрейм и выдают его на линию DMX512 через UART и RS-485 трансивер.

Требования к железу
ESP32 (любая плата с поддержкой Arduino и ESP-NOW) — для мастера и слейвов.

Ethernet модуль W5500 (или W5100) с SPI интерфейсом — для мастера.

RS-485 трансивер (например, MAX485, SN75176) — для каждого слейва.

Блоки питания 5В.

Соединительные провода, разъемы XLR-3 (для DMX).

Настройка мастер-узла
Все параметры задаются в начале прошивки (ArtNetESPNOW_Master.ino):

cpp
// MAC-адрес Ethernet-контроллера (можно оставить как есть)
byte mac[] = { 0xDE, 0xAD, 0xBE, 0xEF, 0xFE, 0xED };

// Статический IP мастер-узла
IPAddress ip(2, 0, 0, 1);
IPAddress subnet(255, 0, 0, 0);
// Шлюз и DNS не обязательны для Art-Net broadcast
Убедитесь, что мастер и компьютер со световым ПО находятся в одной подсети (например, 2.x.x.x).

Мастер автоматически отвечает на ArtPoll, поэтому он будет виден в списке устройств вашего софта.

Настройка слейв-узла
Основные настройки в начале прошивки (ArtNetESPNOW_Slave.ino):

cpp
#define DMX_TX_PIN       17     // Пин для DMX выхода (UART TX)
#define DEFAULT_UNIVERSE 0      // Юниверс по умолчанию
Смена юниверса возможна через Serial Monitor:

Подключитесь к слейву через USB (скорость 115200).

Отправьте число (0–15) и нажмите Enter — юниверс сохранится в энергонезависимой памяти (NVS) и будет использоваться после перезагрузки.

Протокол ESP-NOW
Пакеты ESP-NOW имеют пользовательский тип (первый байт). Используются следующие типы:

Тип пакета	Код	Направление	Описание
PKT_REGISTER	0x01	Slave → Master	Регистрация слейва (передаёт юниверс и имя)
PKT_ACK	0x02	Master → Slave	Подтверждение регистрации, MAC мастера
PKT_DMX_CHUNK	0x03	Master → Slave	Фрагмент DMX-данных (до 200 байт)
PKT_PING	0x04	Master → Slave	Проверка активности
PKT_PONG	0x05	Slave → Master	Ответ на пинг
Структура PKT_DMX_CHUNK
text
[0]     type          (1 байт)  = 0x03
[1-2]   universe      (2 байта) little-endian
[3-4]   offset        (2 байта) смещение в DMX буфере
[5-6]   length        (2 байта) длина данных в этом чанке (≤200)
[7-10]  sequence      (4 байта) порядковый номер фрейма (для защиты от повторов)
[11+]   dmxData       (length байт)
Полный DMX фрейм (512 каналов) передаётся тремя чанками:

offset = 0, length = 200

offset = 200, length = 200

offset = 400, length = 112

Управление подключениями
Слейв при старте шлёт broadcast регистрацию, мастер отвечает ACK и добавляет слейв в реестр.

Каждые 2 секунды мастер шлёт PING всем слейвам. Если слейв не ответил PONG в течение 8 секунд, он удаляется из реестра.

Слейв, не получавший PING более 10 секунд, сбрасывает регистрацию и начинает поиск мастера заново.

Если на один юниверс зарегистрировано несколько слейвов, мастер автоматически переключается на broadcast для экономии эфира.

Прошивка устройств
Установите Arduino IDE и добавьте поддержку ESP32 (https://github.com/espressif/arduino-esp32).

Скачайте или склонируйте репозиторий проекта.

Откройте:

Для мастера: ArtNetESPNOW_Master/ArtNetESPNOW_Master.ino

Для слейва: ArtNetESPNOW_Slave/ArtNetESPNOW_Slave.ino

Убедитесь, что выбрана правильная плата ESP32 и порт.

При необходимости измените параметры (пины, IP и т.д.).

Загрузите прошивку в каждое устройство.

Использование со световым ПО
Любое программное обеспечение, поддерживающее Art-Net output, может отправлять данные мастеру. Проверено с:

QLC+ (Windows, macOS, Linux) — бесплатно

FreeStyler (Windows) — бесплатно

MagicQ onPC (Windows, macOS, Linux)

grandMA 3 onPC (Windows, macOS)

Настройки в софте:

Целевой IP = IP мастер-узла (например, 2.0.0.1)

Порт = 6454

Юниверс = тот же, что настроен на слейвах (или используется в мастере для маршрутизации)

Мастер транслирует только тот юниверс, который пришёл в Art-Net — он не изменяет номер юниверса, поэтому слейвы должны быть настроены на соответствующий номер.

Известные ограничения
ESP-NOW работает только в диапазоне 2.4 ГГц, дальность до 200 м на открытом пространстве (зависит от условий).

Максимальный размер ESP-NOW пакета — 250 байт, поэтому DMX разбивается на чанки.

Все устройства должны работать на одном Wi-Fi канале (в прошивке задан ESPNOW_BROADCAST_CHANNEL = 1).

Одновременно может быть активно до 20 слейвов (ограничение ESP-NOW peer count).

Слейв может слушать только один юниверс. Для разных юниверсов нужны разные слейвы.
