# 🎛 ArtNet ESP-NOW: беспроводная DMX система

> Беспроводная передача DMX512 через ESP32 — Art-Net по Ethernet на входе, ESP-NOW по воздуху, DMX512 на выходе.

---

## Как это работает

```
┌─────────────────────────┐
│   Световое ПО           │  QLC+, FreeStyler, MagicQ...
│   (Art-Net output)      │
└────────────┬────────────┘
             │ Art-Net UDP : 6454
             ▼
┌─────────────────────────┐
│   ESP32 MASTER          │  Ethernet (W5500)
│   + W5500               │  ← слушает Art-Net
└────────────┬────────────┘  → рассылает ESP-NOW
             │
             │  ESP-NOW (Wi-Fi 2.4 ГГц, до 200 м)
     ┌───────┼───────┐
     ▼       ▼       ▼
  SLAVE 1  SLAVE 2  SLAVE N
     │
     │ UART → RS-485
     ▼
  DMX512
  [Световые приборы]
```

Мастер слушает Art-Net и пересылает DMX-данные слейвам по ESP-NOW. Слейвы собирают фрейм и выдают физический DMX512 через RS-485.

---

## Требования к железу

| Устройство | Компоненты |
|---|---|
| **Мастер** | ESP32 + Ethernet-модуль W5500 (или W5100) по SPI |
| **Слейв** | ESP32 + RS-485 трансивер (MAX485, SN75176 или аналог) |
| **Питание** | Блоки питания 5В |
| **Разъёмы** | XLR-3 (для DMX-выхода каждого слейва) |

## Подключение оборудования

### Мастер — ESP32 + W5500

| W5500 | ESP32 |
|---|---|
| CS | GPIO 5 |
| RST | GPIO 26 |
| MOSI | GPIO 23 |
| MISO | GPIO 19 |
| SCK | GPIO 18 |
| 3.3V | 3.3V |
| GND | GND |

> Схема питания: [`ArtNet-Master/docs/esp32_w5500_power.drawio.svg`](ArtNet-Master/docs/esp32_w5500_power.drawio.svg)

### Слейв — ESP32 + RS-485

| RS-485 модуль | ESP32 | Описание |
|---|---|---|
| DI | GPIO 17 | Данные UART → RS-485 |
| DE + RE | GPIO 4 | Управление направлением |
| A | XLR pin 3 | Дифференциальная пара (Data +) |
| B | XLR pin 2 | Дифференциальная пара (Data –) |
| VCC | 3.3V / 5V | Питание (по модулю) |
| GND | GND | Общая земля |

> ⚠️ **Никогда не подключайте ESP32 напрямую к DMX-линии** без RS-485 трансивера — это повредит устройство.

> Схема питания: [`ArtNet-Slave/docs/esp32_rs485_dmx_slave.drawio.svg`](ArtNet-Slave/docs/esp32_rs485_dmx_slave.drawio.svg)

**Распайка XLR-3 для DMX512:**

| RS-485 | DMX-сигнал | XLR pin |
|---|---|---|
| GND | Земля / экран | 1 |
| B | Data – | 2 |
| A | Data + | 3 |

## Настройка прошивок

### Мастер (`ArtNet-Master/src/main.cpp`)

```cpp
byte mac[]       = { 0xDE, 0xAD, 0xBE, 0xEF, 0xFE, 0xED };
IPAddress ip     (2, 0, 0, 1);
IPAddress subnet (255, 0, 0, 0);
```

- Мастер и компьютер со световым ПО должны быть **в одной подсети** (например, `2.x.x.x`).
- Мастер автоматически отвечает на `ArtPoll` — он будет виден в списке устройств вашего софта.

### Слейв (`ArtNet-Slave/src/main.cpp`)

```cpp
#define DMX_TX_PIN       17   // UART TX → RS-485 DI
#define DEFAULT_UNIVERSE  0   // Юниверс по умолчанию (0–15)
```

**Смена юниверса** через Serial Monitor (115200 baud): отправьте число `0`–`15` и нажмите Enter. Значение сохраняется в NVS и переживает перезагрузку.

## Протокол ESP-NOW

### Типы пакетов

| Пакет | Код | Направление | Описание |
|---|---|---|---|
| `PKT_REGISTER` | `0x01` | Slave → Master | Регистрация слейва: юниверс и имя |
| `PKT_ACK` | `0x02` | Master → Slave | Подтверждение: MAC мастера |
| `PKT_DMX_CHUNK` | `0x03` | Master → Slave | Фрагмент DMX-данных (до 200 байт) |
| `PKT_PING` | `0x04` | Master → Slave | Проверка живости |
| `PKT_PONG` | `0x05` | Slave → Master | Ответ на пинг |

### Структура `PKT_DMX_CHUNK`

```
Байты  0      type       (1)  = 0x03
       1–2    universe   (2)  little-endian
       3–4    offset     (2)  смещение в DMX-буфере
       5–6    length     (2)  длина данных (≤ 200)
       7–10   sequence   (4)  защита от повторов
       11+    dmxData         payload
```

Полный DMX-фрейм (512 каналов) передаётся **тремя чанками**:

| Чанк | offset | length |
|---|---|---|
| 1 | 0 | 200 |
| 2 | 200 | 200 |
| 3 | 400 | 112 |

### Управление подключениями

- При старте слейв шлёт **broadcast-регистрацию**; мастер отвечает ACK и добавляет его в реестр.
- Каждые **2 с** мастер шлёт PING всем слейвам. Нет PONG за **8 с** — слейв удаляется из реестра.
- Слейв, не получавший PING более **10 с**, сбрасывается и ищет мастера заново.
- Если на один юниверс зарегистрировано несколько слейвов — мастер автоматически переключается на **broadcast**.

## Совместимое световое ПО

Подходит любое ПО с поддержкой **Art-Net output**:

| ПО | Платформа | Цена |
|---|---|---|
| [QLC+](https://www.qlcplus.org/) | Win / Mac / Linux | Бесплатно |
| [FreeStyler](http://www.freestylerdmx.be/) | Windows | Бесплатно |
| MagicQ onPC | Win / Mac / Linux | Бесплатно (с лимитами) |
| grandMA 3 onPC | Win / Mac | Бесплатно (с лимитами) |

**Настройки в ПО:**

- Целевой IP = IP мастера (например, `2.0.0.1`)
- Порт = `6454`
- Юниверс = тот же, что настроен на слейвах

> Мастер транслирует Art-Net юниверс без изменений — номер юниверса на слейвах должен совпадать.

## Известные ограничения

- **Дальность** ESP-NOW — до 200 м на открытом пространстве (зависит от условий).
- **Размер пакета** ESP-NOW ограничен 250 байтами → DMX делится на 3 чанка.
- **Канал Wi-Fi** должен совпадать у всех устройств (`ESPNOW_BROADCAST_CHANNEL = 1` в прошивке).
- **Максимум 20 слейвов** одновременно (ограничение ESP-NOW peer count).
- **Один слейв = один юниверс** (512 каналов). Для разных юниверсов — разные слейвы.
